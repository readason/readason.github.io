搭建hexo博客

1.项目概述
1.1背景
在互联网时代，个人博客是一个非常流行的方式，用于分享个人见解、学习心得、技术经验或生活感悟。Hexo是一个快速、简单且强大的静态博客框架，基于Node.js构建，支持Markdown格式的文章。
1.2目的
技术分享和展示： 许多技术爱好者和专业人士选择搭建个人博客，以分享他们的技术见解、项目经验和解决问题的方法。Hexo作为一个基于Node.js的静态博客框架，提供了一个简单而高效的方式来创建和管理技术博客。
个人品牌建设： 在互联网上建立个人品牌对于个人职业发展非常重要。通过博客，用户可以展示自己的专业知识、技能和对行业的理解，从而吸引潜在雇主或客户。
学习笔记和总结： Hexo博客也常用于撰写学习笔记和总结。将学到的新知识整理成博客文章不仅有助于加深理解，还能与他人分享，形成学习社区。
文学创作和生活分享： 有些人选择用Hexo博客记录生活琐事、旅行见闻、或进行文学创作。Hexo提供了简单的Markdown编辑方式，让内容创作者可以专注于写作而不必过多关注技术细节。
个人兴趣和爱好： 博客也是一个展示个人兴趣和爱好的好平台。无论是摄影、美食、旅行还是其他兴趣爱好，通过博客分享自己的经验和见解，可以连接同样兴趣的读者。
建立社交影响力： 通过博客在社交媒体上分享有价值的内容，可以建立自己在特定领域的社交影响力。这对于个人品牌建设和职业发展都有积极的影响。
1.3功能及开发环境
1.3.1功能
1.Markdown写作： Hexo使用Markdown语法，使得写作变得简单而直观。Markdown支持轻量级的文本标记，方便写作者专注于内容而非格式。
2.主题选择： Hexo支持丰富的主题，用户可以选择一个符合个人风格的主题，定制博客的外观。一些主题还提供了响应式设计，适应不同设备的显示。
3.代码高亮： 对于技术博客而言，代码高亮是一个重要的功能。Hexo支持多种代码高亮的插件，让代码在文章中更加清晰可读。
4.搜索功能： 通过集成搜索插件，读者能够快速地找到他们感兴趣的内容。
5.社交分享： 方便读者通过社交媒体分享用户的文章，提高博客的曝光度。
1.3.2 开发环境
Node.js和npm： Hexo是基于Node.js的，所以需要安装Node.js和npm。
Hexo CLI： 使用Hexo命令行工具来初始化博客、创建新文章、生成静态文件等操作。安装Hexo CLI：npm install -g hexo-cli。
文本编辑器： 选择一个合适的文本编辑器，推荐使用支持Markdown语法高亮的编辑器，本人选择使用VSCode编辑器。
Git： 为了方便博客的版本管理和部署，使用Git进行代码的版本控制。
GitHub： 如果计划使用GitHub Pages进行博客的托管，需要有一个GitHub
账号。
Hexo主题和插件： 根据个人喜好选择Hexo主题和插件，可以在Hexo官方网站找到一些主题，而插件可以在npm上搜索。本人选择 hexo-theme-matery-develop主题
2.项目原理与技术方法
2.1Git
Git是一个版本控制系统，用于追踪文件和代码的变化。它允许多人协同工作，在项目的不同版本之间进行切换，查看修改历史，并且在需要时能够恢复到先前的状态。
在搭建Hexo博客过程中，Git被用于几个关键的方面：版本控制： Hexo本身生成的静态文件是根据用户的内容和主题生成的。使用Git可以追踪这些文件的变化。这样，如果用户对博客进行了修改，可以通过Git管理这些变更，并且随时可以回退到之前的版本。备份和恢复： Git可以帮助用户备份博客的内容。通过将博客文件存储在Git仓库中，用户可以确保文件的安全性，并在需要时恢复到之前的任何一个状态。部署到GitHub： Git是将Hexo生成的静态文件推送到GitHub仓库的工具。这允许用户轻松地将博客内容上传到GitHub，并通过GitHub Pages服务将其托管在互联网上。Git提供了版本控制和协作功能，使得管理Hexo博客的文件变得更加简单和可靠。通过Git，用户可以追踪博客内容的变化，备份数据，并将博客部署到线上托管平台，如GitHub。
2.2Node.js
Node.js是一个基于Chrome V8引擎的JavaScript运行时环境，用于在服务器端运行JavaScript代码。它允许开发者使用JavaScript编写后端应用程序，而不仅仅局限于前端开发。Node.js采用了事件驱动、非阻塞I/O模型，使得它在处理大量并发连接时表现出色。
2.3Hexo
Hexo的原理是将作者提供的Markdown文档、主题、配置信息等，通过Hexo命令行工具生成静态HTML文件，并可以通过各种部署方式将这些文件发布到互联网上。这种静态网站生成的方式具有简单高效、易于维护和安全可靠的特点。
静态网站生成器： Hexo是一款静态网站生成器，它将Markdown格式的文本文件和一些配置信息转换成静态HTML文件。与动态网站不同，静态网站生成器在构建时将所有页面提前生成，访问者访问时直接获取静态文件，无需动态生成页面内容。这有助于提高网站的访问速度和安全性。
Markdown文档： Hexo使用Markdown格式的文本文件作为博客文章的源文件。Markdown是一种轻量级标记语言，允许作者使用简单的文本标记来编写文章。Hexo将这些Markdown文件解析并转换成HTML。
主题和模板： Hexo支持使用主题来定制博客的外观。主题是一个包含样式表、模板文件和其他资源的目录。Hexo通过主题来渲染生成最终的HTML页面。用户可以选择现成的主题，也可以自定义主题以满足个人需求。
配置文件： 用户可以通过Hexo的配置文件来设置博客的各种参数，如博客的标题、描述、作者信息、主题选择等。配置文件使用YAML格式，用户可以在其中指定Hexo的行为和外观。
2.4Github部署
GitHub Pages是GitHub提供的一个免费静态网页托管服务，它可以从GitHub仓库中的特定分支或文件夹中托管静态文件。
Hexo生成的静态文件就是我们需要托管的内容，通过将这些文件推送到GitHub仓库，我们利用GitHub Pages服务将这些文件发布到互联网上。
GitHub Pages提供了一个简便的方式，通过将Hexo生成的静态文件推送到GitHub仓库，你可以在很短的时间内将自己的博客部署到互联网上，而无需自己搭建服务器。
3.项目实现过程
3.1Hexo初步搭建
1.安装Git
为了把本地的网页文件上传到github上面去，需要用到工具—Git。Git是目前世界上最先进的分布式版本控制系统，可以有效、高速的处理从很小到非常大的项目版本管理。
到git官网上下载.exe文件,Download git

安装成功即可打开Bash进行命令执行

2.安装node.js
Hexo是基于node.js编写的，所以需要安装一下node.js和里面的npm工具。

3.安装Hexo
先创建一个文件夹MyBlog，用来存放自己的博客文件，在这个文件夹下直接右键git bash打开。输入npm install -g hexo-cli安装Hexo。

本地网站配置好后，输入hexo g生成静态网页，然后输入hexo s打开本地服务器。

3.2GitHub部署
1.注册Github账号创建个人仓库

2.生成SSH添加到GitHub
生成SSH添加到GitHub，连接Github与本地。
右键打开git bash，然后输入下面命令：
git config –global user.name “readason”
git config –global user.email “2967708282@qq.com“
然后创建SSH
ssh，简单来讲，就是一个秘钥，其中，id_rsa是你这台电脑的私人秘钥，不能给别人看的，id_rsa.pub是公共秘钥，可以随便给别人看。把这个公钥放在GitHub上，这样当你链接GitHub自己的账户时，它就会根据公钥匹配你的私钥，当能够相互匹配时，才能够顺利的通过git上传你的文件到GitHub上。

将输出的内容复制到框中，点击确定保存。

打开github，在头像下面点击settings，再点击SSH and GPG keys，新建一个SSH，名字随便取一个都可以，把你的id_rsa.pub里面的信息复制进去。

在git bash输入ssh -T git@github.com

3.将hexo部署到GitHub
这一步将hexo和GitHub关联起来，也就是将hexo生成的文章部署到GitHub上，打开博客根目录下的_config.yml文件，这是博客的配置文件，在这里可以修改与博客配置相关的各种信息。

需要先安装deploy-git ，也就是部署的命令,这样你才能用命令部署到GitHub
npm install hexo-deployer-git –save
hexo clean
hexo generate
hexo deploy
hexo clean清除之前生成的东西。 hexo generate顾名思义，生成静态文章，可以用 hexo g缩写 ，hexo deploy部署文章，可以用hexo d缩写
这样在网页输入readason.github.io就可以看到搭建好的个人博客了。

3.3写文章及发布
首先在博客根目录下右键打开git bash，安装一个扩展
npm i hexo-deployer-git。
输入hexo new post “我的第一篇博客”，新建一篇文章。
打开D:\Hexo\source_posts的目录，可以发现下面多了一个文件夹和一个.md文件，一个用来存放图片等数据，另一个则是文章文件。
可以会直接在vscode里面编写markdown文件，可以实时预览，也可以用用其他编辑md文件的软件的工具编写。
编写完markdown文件后，根目录下输入hexo g生成静态网页，然后输入hexo s可以本地预览效果，最后输入hexo d上传到github上。这时打开github.io主页即能看到发布的文章。

3.4主题选择及优化
此次博客搭建选用hexo-theme-matery-develop主题，其特性有：
简单漂亮，文章内容美观易读。butterfly 设计。响应式设计，博客在桌面端。平板、手机等设备上均能很好的展现。首页轮播文章及每天动态切换 Banner 图片。瀑布流式的博客文章列表（文章无特色图片时会有 24 张漂亮的图片代替）。时间轴式的归档页。词云的标签页和雷达图的分类页。丰富的关于我页面（包括关于我、文章统计图、我的项目、我的技能、相册等）。可自定义的数据的友情链接页面。支持文章置顶和文章打赏。支持 MathJax。TOC 目录。可设置复制文章内容时追加版权信息。可设置阅读文章时做密码验证。Gitalk、Gitment、Valine 和 Disqus 评论模块（推荐使用 Gitalk）。集成了不蒜子统计、谷歌分析（Google Analytics）和文章字数统计等功能。支持在首页的音乐播放和视频播放功能
